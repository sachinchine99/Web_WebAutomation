# ShopNest UI Automation Framework

Enterprise-grade Selenium + TestNG automation suite for the ShopNest QA Assignment.  
**Target:** [https://the-internet.herokuapp.com](https://the-internet.herokuapp.com)

---

## Tech Stack

| Layer | Technology |
|---|---|
| Language | Java 17 |
| Browser Automation | Selenium WebDriver 4.21 |
| Driver Management | WebDriverManager 5.8 |
| Test Framework | TestNG 7.10 |
| Build Tool | Maven 3.9+ |
| Reporting | Extent Reports 5 + Allure 2.27 |
| Logging | Log4j2 |
| CI | GitHub Actions |
| Design Pattern | Page Object Model (POM) |

---

## Prerequisites

- Java 17+ installed (`java -version`)
- Maven 3.9+ installed (`mvn -version`)
- Chrome or Edge browser installed
- Git

---

## Quick Start (clone → run in under 5 minutes)

bash
# 1. Clone
git clone https://github.com/<your-username>/shopnest-ui-automation.git
cd shopnest-ui-automation

# 2. Run full suite (Chrome, headed)
mvn clean test

# 3. Run headless (CI mode)
mvn clean test -Dheadless=true

# 4. Run on Edge
mvn clean test -Dbrowser=edge

# 5. Run only smoke tests
mvn clean test -Dgroups=smoke

# 6. Run only regression tests
mvn clean test -Dgroups=regression

# 7. Generate Allure report
mvn allure:report
open target/site/allure-maven-plugin/index.html

## Configuration

All settings live in `src/test/resources/config/config.properties`.

| Key | Default | Description |
|---|---|---|
| `browser` | `chrome` | `chrome` or `edge` |
| `headless` | `false` | Set `true` for CI |
| `explicit.wait` | `15` | Seconds for explicit waits |
| `retry.count` | `1` | Retry attempts on failure |
| `screenshot.on.failure` | `true` | Auto-screenshot on test fail |

**CI override:** Any key can be overridden via environment variable.  
Example: `VALID_PASSWORD` env var overrides `valid.password` config key.

## CI Pipeline

Push to `main` or `develop` → GitHub Actions triggers automatically:

1. Sets up Java 17 + Chrome
2. Runs full TestNG suite in headless mode
3. Generates Allure and Extent reports
4. Uploads reports + screenshots as build artifacts
5. Publishes pass/fail summary in the PR

---

## Parallel Execution

`testng.xml` is configured with `parallel="classes" thread-count="2"`.  
`DriverManager` uses `ThreadLocal<WebDriver>` — each thread has its own isolated driver instance. No shared state.
